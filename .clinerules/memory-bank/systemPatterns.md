# System Patterns: TalkingYar

## Архитектура системы
- Простая модульная структура
- Отдельные модули для разных областей ответственности
- Минимальная связанность между компонентами

## Основные компоненты
1. **Bot Client** - основной клиент подключения к Discord
2. **Voice Manager** - управление подключением и работой в голосовых каналах
3. **Speech Synthesizer** - синтез текста в речь с использованием Gradio API
4. **Audio Player** - воспроизведение и управление аудио потоками
5. **Command Handler** - обработка пользовательских команд
6. **Config Manager** - загрузка и управление конфигурациями

## Паттерны проектирования
- Используем событийную модель discord.js
- Простые функции вместо классов где это возможно
- Явные зависимости между модулями
- Очередь для последовательного воспроизведения сообщений

## Критические пути выполнения
1. Подключение бота -> Аутентификация -> Готовность к работе
2. Получение команды -> Валидация -> Выполнение действия -> Ответ пользователю
3. Подключение к голосовому каналу -> Установка соединения -> Воспроизведение аудио
4. Получение сообщения -> Синтез речи -> Добавление в очередь -> Воспроизведение
